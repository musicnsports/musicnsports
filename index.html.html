Developer Guide: Upgrade to Supabase

Current: All data in localStorage. Auth, posts, likes, comments, follows work client-side only.

Supabase Upgrade Path:

-- profiles table
create table profiles (
  id uuid references auth.users primary key,
  username text unique not null,
  bio text,
  avatar_url text,
  created_at timestamp with time zone default now()
);
alter table profiles enable row level security;
create policy "Public profiles visible" on profiles for select using (true);
create policy "Users can update own" on profiles for update using (auth.uid() = id);

-- posts, likes, comments, follows tables follow same pattern
-- Replace localStorage calls with: supabase.from('posts').select()

Swap localStorage.getItem() → supabase.from('table').select(). Auth: supabase.auth.signUp(), signInWithPassword(). RLS ensures users only edit their own data.
